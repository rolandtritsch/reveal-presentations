What is the problem?

  • Identity
  • Double spending
  • Identity and double-spending
  • Best explained with transfering real money ... 1 dollar ... from me to you
  • Doing this on a large scale with "virtual" or "electronic" money requires an intermediary
  • These intermediaries end up with a lot of power (and charge a lot, but ... not really)

How is blockchain technology solving these problems?

  • Identity - Easy
  • Double spending - Not really
  • Easy and not really
  • Easy ... by using private/public key pairs to establish identity
  • Not really ... because the blockchain does not really prevent double-spending, but gurantees that it will get detected and that exacly one spend will be selected to be the correct/true one
  • Blockchain technology is a tool of distributed consensus (identity, ownership (of a thing), establishing facts)
  • A blockchain is a globally shared, distributed, transactional database

Concepts

Transaction

  • Transfering *ownership* of something from somebody to somebody else
  • Create a transaction. Sign it with your private key.

Concepts

Blocks & Blockchains

  • The process of building the blockchain is called mining
  • The miners receive all transactions and will the check the blockchain to make sure you have sufficient funds to make the transaction
  • All valid transactions are put into a block. The miner then needs to calc the hash of the block
  • How that hash looks like can be influenced by putting a nounce into the block
  • That nounce will be changed (incremented) until a hash with the right properties will be generated
  • That process is very CPU intensive. Right now the miners own compute power equal to the combined power of the 500 biggest super-computers in the world
  • With that in mind the blockchain is considered correct, because it is based on the Proof-of-Work (means it is proven that you have done the work to build it)
  • If somebody want to attack the blockchain, you must have and spend more CPU power than the miners over an extended period of time to build blocks (with malicious transactions) faster then the miners

Concepts

Distributed Ledger

  • Blockchains are viewed/talked about as globally, distributed ledgers
  • And they are ... and they are not, because ... to build a ledger (a list of accounts and balances you actually have to travers the entire list of transactions (backwards)
  • This is what actually happens when you transfer money. That is what the miners are doing when they build a block. They go backwards until they find enough money to put your transaction into the block and then they solve the block

Ethereum

Blockchain on Steroids

  • Problem is ... by now it has become very hard to horseshoe all usecases into the initial implementation (the one that runs bitcoins)
  • Meet Ethereum .. you can call it blockchain on steroids
  • But ... it is just another implementation of a blockchain ... running a VM (means you can run code on the nodes)
  • Main change ... adds the concept of a contract
  • Means you can send a transaction to a person or a contract
  • Contracts are snippets of code who can run a transaction (type). Contracts define new transaction types
  • Solves a couple of other problems ... e.g. Proove-of-Work vs. Proove-of-Stake
  • These apps run on a custom built blockchain, an enormously powerful shared global infrastructure that can move value around and represent the ownership of property
